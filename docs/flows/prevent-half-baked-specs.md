## examples
### db-spec
Bitte erstelle in einem canvas eine spezifikation fuer die datenhaltung fuer diese api. Die daten sollen als JSON files auf der platte gehalten werden in den verzeichnissen * data/managed/todos * data/managed/reminders die daten werden beim starten der anwendung von den jsaon files geladen und dann komplett in memory bearbeitet. Es gibt ein konfigurationsfile in * data/managed/meta welches regelt unter welcehn umstaenden die daten ins filesystem geschriebne werden. Folgende optionen sollen moeglich sein: * json files nach jeder modifikation im file system updaten * oder json files im hintergrund alle N minuten neu schreiben wenn es sich geaendertr hat * oder json files nur mit einem expliziten api call neu schreiben. Ist das was ich beschreibe klar verstaendlich und konsistent? wenn noch irgendetwas unklar ist bitte beschprechen wir das jetzt bevor du den canvas erstellst.
