Why are the small julia sets in the kohonen map so blury? If I have feature width and height set to 200 it still appears super blury. Before you change any code please explain the root cause of this bug.

```
• The root cause is that the map thumbnails are always rendered into a very small fixed display size, regardless of your feature resolution.

  Two concrete reasons:

  1. Each SOM cell is hard-coded to 28 pixels.
     In the map renderer, MAP_CELL_SIZE is fixed at 28 in code/v001/packages/web/src/canvas/som-map-canvas.tsx:11. Every prototype image, even if it is 200 x 200, gets drawn into a 28 x 28 destination rectangle in code/v001/packages/web/src/
     canvas/som-map-canvas.tsx:130 and code/v001/packages/web/src/canvas/som-map-canvas.tsx:157.
  2. The browser is explicitly smoothing that downsampled image.
     The renderer sets context.imageSmoothingEnabled = true before drawImage(...) in code/v001/packages/web/src/canvas/som-map-canvas.tsx:54. So the browser resamples your 200 x 200 prototype down to 28 x 28 with interpolation, which produces
     the soft/blurry look.

  There is also a second-level constraint: the whole map canvas is styled with width: 100% and max-height: 480px in code/v001/packages/web/src/styles/app.css:314 and code/v001/packages/web/src/styles/app.css:321. So even if the internal canvas
  is large, the UI may still visually compress it.

  So increasing featureWidth/featureHeight improves the prototype data, but it does not make the tiny map thumbnails sharper by itself, because the app still crushes that data into a 28-pixel cell and smooths it.
```
